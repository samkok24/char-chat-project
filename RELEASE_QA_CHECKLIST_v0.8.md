## v0.8 릴리즈 검수 체크리스트 (v0.7.4 → v0.8 대규모 업데이트)

목적: 운영/데모에서 “화이트스크린/필터 오동작/모달 UX 붕괴/채팅 시작 실패” 같은 치명 이슈를 사전에 제거한다.  
범위: **위저드(CreateCharacterPage)**, **30초 생성 모달(QuickMeetCharacterModal)**, **RP/시뮬/커스텀 개념(캐릭터 타입)**, **태그/필터**, **CMS(구좌/태그 관리)**, **채팅/인라인 이미지**.

---

## 사전 준비(데이터/계정)

- [ ] **관리자 계정**: CMS 접근 가능
- [ ] **일반 유저 계정**: 캐릭터 생성/채팅 가능
- [ ] **테스트 캐릭터 최소 5종**
  - [ ] 일반 캐릭터챗(롤플) 1개 이상
  - [ ] 일반 캐릭터챗(시뮬) 1개 이상
  - [ ] 일반 캐릭터챗(커스텀) 1개 이상
  - [ ] 원작챗 캐릭터 1개 이상
  - [ ] 웹소설(스토리) 1개 이상(탐색/구좌 혼합 확인용)
- [ ] **오프닝 2개 이상**인 캐릭터 1개(오프닝 선택→새로대화 검증)
- [ ] **인라인 이미지 코드** 준비(예: `[[img:https://...]]` 또는 `{{img:https://...}}`)

---

## P0 (무조건 통과: 운영/데모 치명도)

### 1) 홈/탐색/캐릭터 탭 기본 동작

- [ ] **홈 진입** 시 화면이 깨지지 않는다(로딩 스켈레톤 후 정상 렌더)
- [ ] **캐릭터 탭 태그칩** 상단에 `모두`가 보인다(기존 “전체” 문구가 남아있지 않음)
- [ ] `모두` 옆에 **`롤플 / 시뮬 / 커스텀`** 칩이 고정으로 보인다
- [ ] `롤플` 클릭 시 목록이 **롤플 캐릭터만**으로 필터된다
- [ ] `시뮬` 클릭 시 목록이 **시뮬 캐릭터만**으로 필터된다
- [ ] `커스텀` 클릭 시 목록이 **커스텀 캐릭터만**으로 필터된다
- [ ] 다시 `모두` 클릭 시 필터가 초기화된다

### 2) 격자 카드(CharacterCard) 표기

- [ ] 격자 카드에 **턴수 배지**가 숫자로 보인다(불필요한 `∞` 고정 표기 없음)
- [ ] 격자 카드 태그가 **줄바꿈(wrap)되며 “쭉쭉” 노출**된다(상단 몇 개만 나오고 끊기지 않음)
- [ ] 격자 카드에 **남성향/여성향 + 롤플/시뮬/커스텀** 태그가 함께 보인다

### 3) 상세/모달 UI(캐릭터)

- [ ] 모바일 상세 모달에서 **“캐릭터” 뱃지 대신 턴수 뱃지**가 보인다
- [ ] 모달의 태그는 **공개일|수정일 아래**에 표시된다
- [ ] 모달/상세페이지의 태그 디자인이 **격자 카드와 톤이 일관**된다

### 4) 오프닝 칩(상세/모달) + “새로 대화” 동작

- [ ] 오프닝이 1개여도 **오프닝명 칩이 노출**된다
- [ ] 오프닝 칩을 클릭하면 **첫상황/첫대사 프리뷰가 즉시 해당 오프닝으로 변경**된다
- [ ] 두 번째 오프닝 선택 후 **“선택한 오프닝으로 새로 대화”**를 누르면:
  - [ ] 실제 채팅이 두 번째 오프닝의 첫상황/첫대사로 시작한다
- [ ] 오프닝 이름을 변경(예: `easy모드`)하면 모달/상세페이지 칩 라벨에 **즉시 반영**된다

### 5) 채팅 UI: 인라인 이미지 렌더링

- [ ] 채팅 말풍선에서 `[[img:...]]/{{img:...}}`가 **이미지로 렌더링**된다
- [ ] 엔딩 조건/에필로그(실제 채팅 출력)에서도 **이미지 렌더링**이 동일하게 동작한다

### 6) 위저드(CreateCharacterPage) 핵심 플로우

- [ ] RP/시뮬/커스텀 모드 전환이 정상 동작한다(선택/저장/재진입 시 유지)
- [ ] 토글 스위치의 **동그라미(thumb)가 실제로 좌↔우로 이동**한다(배경색만 바뀌는 현상 없음)
- [ ] “작품명 구체적으로” 토글:
  - [ ] ON일 때 라벨이 **작품명 구체적으로**
  - [ ] OFF(기본값)일 때 라벨이 **제목 스타일 자유**
  - [ ] 위치가 “빠르고 트렌디하게 생성” **바로 아래**에 있다
  - [ ] 시뮬 모드에서는 **숨김 처리**된다
- [ ] 설정메모 트리거:
  - [ ] “추가”를 누르면 빈 입력 줄이 **실제로 생성**된다
  - [ ] 입력 후 저장/재진입 시 값이 유지된다
  - [ ] 적용 대상 드롭다운에 **오프닝 이름(예: 오프닝1/easy모드)** 이 표시된다(기본설정1 표시 금지)
- [ ] 턴수별 사건:
  - [ ] “필수 지문/필수 대사”에 이미지 코드를 넣으면 **미리보기에서 인라인 이미지로 보인다**

### 7) 30초 생성 모달(QuickMeetCharacterModal)

- [ ] 토글 thumb 이동 애니메이션이 정상 동작한다
- [ ] 생성 완료 후 캐릭터가 정상 저장되고 상세/채팅 진입이 된다
- [ ] 콘솔에 런타임 크래시(예: const 재할당류)가 없다

### 8) 스토리 에이전트 네비/헤더

- [ ] 사이드바에 “스토리 에이전트” 버튼이 **대화내역 아래**에 보인다
- [ ] `/agent` 상단 왼쪽에 **`< 메인으로 가기`** 링크가 있고, `<`가 중복되지 않는다

### 9) CMS(관리자) — 구좌/태그

- [ ] CMS > 구좌 조작:
  - [ ] **활성화(n) / 비활성화(n)** 보기 탭이 보인다
  - [ ] 각 탭에서 위/아래 버튼으로 **순서 변경이 “바로 눈에 보이게”** 동작한다
  - [ ] 구좌 콘텐츠 선택 모달에서 검색 결과가 많아도 **하단 취소/적용 버튼이 항상 보인다**
- [ ] CMS > 구좌 콘텐츠 선택 모달:
  - [ ] 태그 입력에 `롤플` 또는 `#롤플`을 넣어도 정상 동작한다
  - [ ] `전체` 타입 + `#롤플` 태그 + 검색 시 캐릭터 결과가 나온다
- [ ] CMS > 태그 관리:
  - [ ] `롤플/시뮬/커스텀`이 “전체 태그” 목록에 나타난다
  - [ ] 숨김/상단우선 노출 설정이 저장 후 실제 캐릭터 탭에 반영된다

## P1 (중요: 사용자 불만/CS 유발)

- [ ] PC 웹에서 “지금 대화가 활발한 고유 캐릭터” 섹션은 **더보기만** 보인다(헤더 좌우 화살표 없음)
- [ ] 각 페이지 전환(Home/Detail/Chat/Create/CMS/Agent)에서 **브라우저 콘솔 에러가 없다**
- [ ] 검색/필터 조합(검색어 + 태그 + 타입)에서 결과가 0개일 때도 UI가 깨지지 않는다
- [ ] 모바일에서 모달/시트/드롭다운이 화면 밖으로 튀지 않는다(특히 CMS 모달/위저드 드롭다운)

## 운영 확인(권장)

- [ ] 운영 서버 로그에서 500/422가 급증하지 않는다(캐릭터 생성/태그/ CMS 저장)
- [ ] 캐릭터 생성/채팅 시작 실패 시 사용자에게 안내가 노출된다(무응답/무한 로딩 금지)
