## Q1. 웹소설 콘텐츠를 어떻게 할까?

현재 홈에 "웹소설" 탭과 원작챗(웹소설 기반 캐릭터) 섹션이 있지만,  
아직 웹소설 작품이 거의 올라오지 않은 상태입니다.  
작품 없이 빈 탭이 노출되면 서비스가 미완성으로 보일 수 있습니다.

- [ ] **A) 완전 숨김** — 웹소설 탭, 원작챗 구좌 모두 비노출. 깔끔하게 캐릭터챗만 보여주자
- [ ] **B) 탭만 숨기고 원작챗은 유지** — 탭은 제거하되, 이미 만든 원작챗 캐릭터는 일반 캐릭터 목록에 섞어서 노출
- [ ] **C) 탭은 남기되 "준비중" 표시** — "웹소설 곧 오픈!" 같은 티저 문구로 기대감 유지
- [ ] **D) 기타**: _______________

> **참고 (대표 의견)**: A안 추천. 지금은 캐릭터챗의 완성도를 보여주는 게 우선이고, 빈 탭은 이탈 원인이 됨. 웹소설은 작품이 3개 이상 확보되면 그때 다시 열어도 늦지 않음.

## Q2. 웹소설을 숨기면, 기존 문구/메뉴는 어떻게 바꿀까?

현재 사이드바, 홈 헤더 등에 "웹소설", "스토리" 같은 문구가 남아있습니다.

- [ ] **A) "캐릭터"로 통일** — 메뉴/구좌 타이틀 등 모든 곳에서 "웹소설/스토리" 문구 제거, "캐릭터"로 통일
- [ ] **B) "캐릭터 & 스토리"** — 웹소설 탭은 숨기되, 브랜딩 차원에서 "스토리" 워딩은 남기기 (향후 웹소설 복귀 대비)
- [ ] **C) 그냥 탭만 숨기고 문구는 건드리지 말자** — 최소한의 수정만
- [ ] **D) 기타**: _______________

> **참고 (대표 의견)**: A안 추천. "스토리"가 남아있으면 유저가 찾다가 없어서 혼란스러워함. 다시 열 때 문구 추가하는 건 30분이면 됨.

## Q3. 웹소설 탭 자리에 다른 탭이 필요할까?

웹소설 탭을 숨기면 홈에 "캐릭터" 관련 콘텐츠만 남습니다.

- [ ] **A) 탭 없이 캐릭터만** — 탭 구분 자체가 필요 없음. 캐릭터 목록 + 태그 필터로 충분
- [ ] **B) "인기" / "신규" 탭** — 인기 캐릭터 / 최신 캐릭터를 탭으로 분리
- [ ] **C) "롤플" / "시뮬" 탭** — 모드별로 탭 분리 (태그 필터와 중복될 수 있음)
- [ ] **D) "추천" 탭 추가** — AI 추천 또는 운영자 큐레이션 탭 하나 추가
- [ ] **E) 기타**: _______________

> **참고 (대표 의견)**: A안 추천. 이미 태그칩(롤플/시뮬/커스텀 + 장르)으로 필터링이 가능해서, 탭을 추가하면 오히려 동선이 복잡해짐. 가입자 10명 단계에서는 심플한 게 최선.

## Q4. 홈 구좌(섹션) 구성을 어떻게 할까?

현재 구좌에 웹소설 관련 섹션이 섞여 있고, 롤플/시뮬 캐릭터가 따로 구분되지 않습니다.

- [ ] **A) 롤플/시뮬 구좌 분리** — "인기 롤플레잉 캐릭터" / "인기 시뮬레이터" 구좌를 별도로 만들기
- [ ] **B) 모드 구분 없이 통합** — 구좌는 "인기", "신규", "추천" 등 기준으로만 나누고, 롤플/시뮬은 태그 필터에 맡기기
- [ ] **C) 큐레이션 구좌 강화** — 운영자가 직접 뽑은 "이번 주 추천", "입문용 캐릭터" 같은 테마 구좌를 주 1회 이상 교체
- [ ] **D) A + C 병행** — 모드별 자동 구좌 + 운영 큐레이션 구좌 함께 운영
- [ ] **E) 기타**: _______________

> **참고 (대표 의견)**: D안 추천. 롤플/시뮬 구좌를 분리하면 유저가 "이런 종류도 있구나" 발견하게 되고, 큐레이션 구좌는 매주 바뀌면 재방문 동기 부여됨. 단, 캐릭터 수가 적으면 구좌가 비어 보일 수 있으니 초기에는 B로 시작하고 캐릭터 30개 이상 되면 D로 전환하는 것도 방법.

## Q5. 배너를 어떻게 바꿀까?

현재 배너가 이전 버전 기준이라 업데이트 내용을 반영하지 못하고 있습니다.  
곧 GDN(구글 디스플레이 네트워크) 광고도 시작할 예정입니다.

- [ ] **A) 업데이트 소개 배너** — "캐릭터 생성 리뉴얼! 30초면 나만의 캐릭터 완성" 같은 신규 기능 홍보
- [ ] **B) 서비스 소개 배너** — 신규 유저 대상. "AI와 대화하는 새로운 방법, 챕터8" 같은 브랜딩
- [ ] **C) 캐릭터 추천 배너** — 인기 캐릭터 이미지를 활용한 "이 캐릭터와 대화해보세요" 배너
- [ ] **D) 이벤트/프로모션 배너** — "베타 기간 전 서비스 무료!" 같은 가입 유도
- [ ] **E) 복수 선택 — 위 중 어떤 조합?**: _______________

> **참고 (대표 의견)**: GDN으로 들어오는 신규 유저는 챕터8이 뭔지 모름. B(서비스 소개) + D(무료 이벤트)를 메인 배너 2장으로 돌리고, A(업데이트 소개)는 3번째 배너로. C는 추천할 만한 퀄리티 캐릭터가 확보된 이후에.

## Q6. 배너 디자인 톤/분위기는?

- [ ] **A) 애니/일러스트 감성** — 캐릭터 일러스트 중심, 밝고 화려한 컬러
- [ ] **B) 미니멀/모던** — 텍스트 중심 + 단색 배경, 깔끔한 느낌
- [ ] **C) 게임 감성** — 다크 톤 + 네온/그라데이션, "플레이" 느낌
- [ ] **D) 혼합** — 서비스 소개는 미니멀, 캐릭터 추천은 일러스트 감성
- [ ] **E) 기타**: _______________

> **참고 (대표 의견)**: D안. 배너마다 목적이 다르니까 톤도 달라야 함. 서비스 소개는 깔끔하게, 캐릭터 추천은 캐릭터 이미지를 크게 써서 시선을 끌어야 함.

## Q7. "빠르게 캐릭터 만들기" 버튼을 더 강조할까?

30초 생성은 이번 업데이트의 킬러 기능입니다.  
유저가 직접 캐릭터를 만들어보게 해야 리텐션이 올라갑니다.

- [ ] **A) 현재 위치 유지** — 홈에 버튼이 있으니 충분함
- [ ] **B) 배너에서 바로 연결** — 배너 CTA(버튼)를 누르면 30초 모달이 바로 열리게
- [ ] **C) 첫 방문 유저에게 자동 팝업** — 회원가입 직후 "나만의 캐릭터를 만들어보세요!" 팝업
- [ ] **D) 사이드바/하단 고정 버튼** — 어디서든 접근 가능한 플로팅 버튼
- [ ] **E) 기타**: _______________

> **참고 (대표 의견)**: B + C 조합 추천. 신규 유저는 뭘 해야 할지 모르기 때문에, 가입 직후 한 번 자동으로 안내하고, 배너에서도 바로 접근 가능하게 하면 전환율이 올라감. D는 UI가 산만해질 수 있어서 비추.

## Q8. GDN 광고 랜딩 페이지는 어디로?

가입자 10명 상태에서 GDN을 시작합니다.  
광고를 클릭한 사람이 처음 보는 페이지가 중요합니다.

- [ ] **A) 홈 (`/`)** — 서비스 전체를 보여주기
- [ ] **B) 인기 캐릭터 상세페이지** — 바로 대화 시작 유도 (예: 가장 인기 있는 캐릭터)
- [ ] **C) 회원가입 페이지** — 먼저 가입시키고 시작
- [ ] **D) 별도 랜딩 페이지 제작** — "챕터8이란?" 소개 + 캐릭터 미리보기 + 가입 CTA
- [ ] **E) 기타**: _______________

> **참고 (대표 의견)**: 초기에는 A(홈)로 시작하되, 전환율 데이터를 보고 B나 D로 전환. 별도 랜딩은 리소스가 들지만 전환율은 가장 높음. C(회원가입 바로)는 이탈률이 높아서 비추.

이전 버전 기준의 공지, 이벤트가 아직 남아있습니다.

- [ ] **A) 전부 그대로 두기** — 히스토리 보존 차원

- [ ] **D) B + C 동시** — 정리 + 새 이벤트
- [ ] **E) 기타**: _______________
